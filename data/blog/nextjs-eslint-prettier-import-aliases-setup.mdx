## âš™ï¸ Absolute Imports (a.k.a. Import Aliases)

By default, Next.js forces you to write relative imports like:

```tsx
import Button from '../../../components/ui/Button';
```

This gets messy fast. Letâ€™s fix that.

### Step 1: Open `tsconfig.json`

Inside the root of your project, update the config like so:

```json
{
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "@/*": ["./*"]
    }
  }
}
```

> This allows you to use @/ as an alias to the root folder.
> 

### Step 2: Use the Alias

Now you can import files like this:

```tsx
import Button from '@/components/ui/Button';
```

### Optional: Alias Intellisense in VS Code

Make sure your `jsconfig.json` or `tsconfig.json` is properly configured so VS Code doesnâ€™t show errors or squiggles.

## ðŸŽ¯ Prettier Setup (Tailored Formatting + Tailwind Plugin)

To ensure consistent and automatic formatting across the entire codebase, I configure Prettier with rules that match our team style â€” plus a Tailwind plugin to sort class names intelligently.

This helps keep both your JavaScript/TypeScript and CSS utility classes clean, readable, and standardized.

### ðŸ§© Step 1: Install Prettier + Tailwind Plugin

```bash

```

---

### ðŸ“„ Step 2: Create `.prettierrc`

```json
{
  "$schema": "https://json.schemastore.org/prettierrc.json",
  "tabWidth": 2,
  "printWidth": 100,
  "semi": true,
  "useTabs": false,
  "singleQuote": false,
  "bracketSpacing": false,
  "endOfLine": "auto",
  "arrowParens": "always",
  "trailingComma": "all",
  "plugins": ["prettier-plugin-tailwindcss"]
}

```

### ðŸ” What This Does:

| Rule | Description |
| --- | --- |
| `tabWidth: 2` | Use 2 spaces per indentation level |
| `printWidth: 100` | Wrap lines that exceed 100 characters |
| `semi: true` | Always add semicolons |
| `useTabs: false` | Use spaces instead of tabs |
| `singleQuote: false` | Prefer double quotes (common in HTML-like JSX) |
| `bracketSpacing: false` | No spaces inside object literals: `{a:1}` not `{ a: 1 }` |
| `arrowParens: "always"` | Always include parentheses in arrow functions |
| `trailingComma: "all"` | Adds trailing commas where valid in ES5 |
| `plugins: ["prettier-plugin-tailwindcss"]` | Automatically sorts Tailwind utility classes |
